2 Fix #2

Make Tools Load
Only When Needed

Instead of loading every tool schema upfront, Claude remembers just the names and fetches full details only when it needs them. Like a phonebook vs memorizing every phone number.

Without lazy loading

schema loaded
schema loaded
schema loaded
schema loaded
schema loaded

125 tool schemas = all in memory

With lazy loading

name only
name only
full schema (needed!)
name only
name only

Only 1 full schema loaded at a time

.claude/settings.json

{

"env": {

"ENABLE_TOOL_SEARCH": "auto:5"

}

}

What auto:5 means:

If a server has more than 5 tools, only load tool names upfront. Fetch the full schema on demand when Claude actually needs it.

~6,000+ tokens saved

Per turn. The biggest single win.
